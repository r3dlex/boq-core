# Contributing to boq-core

1. Start from an issue, PRD/spec, or documented local fallback in `.ai/work-intake/`.
2. Create a branch and keep changes small, reviewable, and reversible.
3. Run the repo-local checks listed in `.ai/language-packs/language-packs-snapshot.json`.
4. Open a PR and complete the architect + reviewer + executor review loop.
5. Merge only when local CI and GitHub CI are green and all actionable comments are resolved.

Hosted policy changes are not part of normal contribution flow and require explicit admin confirmation.
